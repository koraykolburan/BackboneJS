BackboneJS Documentation

Backbone.Model

Backbone.Model wrapped around our domain data. Essentially every time you need to deal with a series of REST end points for reading or updating data then you need a Backbone.Model.
- Data properties
- URL for REST operations

Backbone.View

Backbone.View can be used to display the content of a model on screen. And how a view can act like a bridge between the user and the model. Reacting the UI Events and on Model Change Events.
- Transfers model to UI
- Handle UI Events

Backbone.Router

However web application always starts with the user typing in a URL in a web browser. The link between the URL and what the user is going to see on the screen is provided by a Backbone.Router .
The router is just that, it maps a URL resource to a page containing views and models to do that Backbone uses URL hash fragments.
- Handle URL change events
- ...and "route" the user to the right View and Model
- example: /myapp#homepage
